테이블 코딩 중요 point

  1. 테이블 높이는 height가 아니라 padding 으로 세로값을 준다. (확장성에 좋다.)
  2. 클레스는 테이블 종류로 class 이름을 정한다
    • table-lest
    • table-view
    이것을 구분해서 스타일을 줘야 한다
  3. 테이블 안에 테이블 sub-table 이 생길 경우는 = table > thead > th {ooo} 처럼 클레스를 지정해주면 된다.
  4. 테이블,th,td 에 css style을 줘도 되지만 thead,tbody,tr에는 부득이한 상황이 생기지 않는다면 스타일을 주지 않는다.
  5. table-layout:fixed : 테이블에지정된 width 값을 고정 시켜ㅑ 준다 * 줄바꿈은 일어나지 않기 때문에 원한다면 그 뒤에 word-break:break-all; 를 선언해준다
  6. 글자가 넘처날때 요약 용도로 이상 넘어가 table에 table-layout:fixed 를 걸어주고 요약하고자 하는 곳에 style을 overflow:hidden; text-overflow:ellipsis; white-space:nowrap 을 주면 테이블에서 넘처나는 글자를 ... 으로 대체, 줄여준다.
  7. table에 tfoot 가 들어가는 경우 thead 보다 tfoot을 더 위에 올려야 접근성에 좋다
  8. table에 어디든 width:100% 를 주고 시작하자

필드

필드를 고려해서 마크업을 해야 할 필요가 있다 필드란 어떤 태그를 어디서든 사용하고 간격, 스타일, position 배정 등 많은 유리함이 있다.
이 경우 float을 해서 떠 버린 코드들도 묶어 줄 수 있어서 css에 claer:both를 쓰지 않고 편리하게 집어 넣는 대로 사용 할 수 있는 단점이 있다.
어떤것이 짝이고 어떤 것이 한 묶음인지를 잘 파악하지 못하면 쓸때없는 태그가 될 가능성이 있다. 예를 들면 class="에어리어"

  • margin-bottom, padding-bottom은 특별한 경우가 아닌 이상 가급적 주지 않는다.(top으로 대체 해서 간격을 준다)

    • reset tag

    리셋 테그에 가끔 하나의 tag를 - ! important - 를 적용시켜 class로 만들어 놓는 경우가 있는데 공통적이지 않고 특별한 경우에 준다. 임폴턴트가 들어가기 때문에 많이 쓰면 용도에 맞춰 쓰기 어렵기 때문에
    가급적이면 자제하자.

    용어

    • 마크업 : HTML 구조를 짜는 것
    • W3Cschool : 크로스브라우징의 허용 태그와 모르는 태그를 검색하기 좋은 사이트

    tip

    • display:block을 줘서 모양을 내는 방법도 있지만 이 코드를 쓰지 않고도 float,position도 속성:block 과 같고 이 tag 들이 들어가면 속성tag를 빼줘야 한다.
    • before와 after 태그를 활용하면 아이콘, style 등 유용하게 쓸 수 있다. 그리고 불필요한 마크업을 확실히 줄여줄 수도 있다.
    • PSD (디자인) 의 총 위드는 min-width 이다 min-width는 지정 값 이하로 줄어들게 하지 않기 하기 위해 쓴다. 즉 PSD는 최소 width 값의 디자인이란걸 알아야 한다.
    • 로케이션에는 가급적이면 a tag를 연속으로 쓰지 않는다.
    • reset css의 내용을 확인하여 불필요한 style이 있다면 지워주자
    • 마크업은 타인들도 봤을때 간편해 보여야 하는것 HTML 태그야 말로 깔끔하고 가능하면 짧게 써야할 필요가 있다.
    • PSD 상의 이미지, 아이콘을 자를때는 주번 다른 layer의 눈을 끄고 shift + ctrl + c 를 눌러서 간편하게 자르자
      • jpg : 이미지들을 확장자로 써준다. (그림 등등) 용량이 중간정도로 적다
      • gif : 곡선 없는 아이콘,블릿 등에 쓴다. 용량이 제일 작다.
      • png : 용량이 제일 많지만 jpg 처럼 화잘이 좋고 투명으로도 자를 수 있어 아이콘 같은 경우에 쓰인다.
    • 폰트를 hover 할때 출렁거리지 않게 한다.
    • a 태그에 href="#" 과 href="#none" 의 차이는 상단으로 올라간다와 올라가지 않는다로 구분된다.
    • PSD 상에는 hover로 만든 디자인과 hover로 만들지 않은 디자인을 한꺼번에 준다. 구분하고 퍼블리셔는 코딩을 해야한다.
    • select 에는 절때 position을 쓰면 안된다 깨질 우려가 크기 때문이다.
    • 개발로 text가 변경 될것 같은 상황에는 span으로 한번 감싸주어 개발자들의 편의를 도와주자
    • display:table을 어미에 주면 자식 tag들은 테이블 속성이 된다.
    • position:absolute로 가운대 정렬이 필요하다면 또 다른 방법으로 left:0; right:0; text-align:center도 다른 방법이다
